Weblet

Self-contained web apps for AI agents

GitHub Specification Examples

The Problem

You ask an AI to build a quick web app, and suddenly you're drowning in React setup, npm dependencies, and webpack configs. All you wanted was a simple app.

And even if you get it working - how do you share it? How does an AI agent launch it with data? How does it fit into an ecosystem of agent skills?

The Solution

A weblet is a self-contained web app in a single folder. No build step. No dependency hell. Just files that work.

Open index.html in a browser, and it runs. Want TypeScript? Use Bun - it transpiles on the fly. Want to share it? Zip the folder.

Agent Integration

Weblets can receive data from AI agents via window.__AGENT_CONTEXT__ and emit events back. They work standalone too - checking for agent context and gracefully degrading when absent.

This makes weblets the UI layer for Agent Skills - the skill provides the brain, the weblet provides the face.

Quick Start

Open Claude Code (or your AI assistant) and say:

Build me a weblet that converts markdown to HTML.
Use browser runtime, no external dependencies.
